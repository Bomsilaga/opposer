Setup Guide
Everything you need to go from demo → live in about 20 minutes. You need a free Supabase account and your Gumroad product set up.
Step-by-step
Follow these in order. Each step links to exactly where to go.
1
Create a free Supabase project at supabase.com. Takes 2 minutes. Choose a region closest to your users (e.g. Sydney, US East, EU West).
2
Run the SQL schema (see below) in your Supabase SQL Editor. Creates the subscribers table and run_log table.
3
Deploy the Edge Function (see below) to your Supabase project via the Supabase CLI: supabase functions deploy gumroad-webhook
4
In Gumroad → Settings → Advanced → Ping URL: paste your Edge Function URL: https://YOUR_PROJECT.supabase.co/functions/v1/gumroad-webhook
5
Paste your Supabase URL and anon key in the Settings tab. The dashboard immediately switches to live data.
6
Add your Anthropic API key as a Supabase secret: supabase secrets set ANTHROPIC_KEY=sk-ant-.... The frontend app should call your Edge Function proxy, not Anthropic directly.
SQL Schema
Run this once in your Supabase SQL Editor to create the tables.
-- Subscribers table CREATE TABLE subscribers ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, email text UNIQUE NOT NULL, name text, tier text NOT NULL DEFAULT 'free', status text NOT NULL DEFAULT 'active', run_cap integer NOT NULL DEFAULT 3, model text NOT NULL DEFAULT 'claude-haiku-4-5-20251001', runs_used_this_month integer DEFAULT 0, billing_month text, gumroad_subscription_id text, gumroad_product_id text, mrr_usd numeric DEFAULT 0, joined_at timestamptz DEFAULT now(), renewal_date timestamptz, last_active timestamptz, notes text ); -- Run log table CREATE TABLE run_log ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, subscriber_id uuid REFERENCES subscribers(id), ran_at timestamptz DEFAULT now(), doc_type text, model_used text, tokens_used integer, is_trial boolean DEFAULT false ); -- Enable RLS ALTER TABLE subscribers ENABLE ROW LEVEL SECURITY; ALTER TABLE run_log ENABLE ROW LEVEL SECURITY; -- Service role has full access -- Anon key: read-only for own row by email CREATE POLICY "own_row" ON subscribers FOR SELECT USING (email = current_setting('request.jwt.claims', true)::json->>'email');
Gumroad Webhook Edge Function
Save as supabase/functions/gumroad-webhook/index.ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2' const TIER_MAP: Record<string, {tier:string, cap:number, mrr:number, model:string}> = { 'opposer-student': {tier:'student', cap:20, mrr:9.99, model:'claude-sonnet-4-6'}, 'opposer-solo': {tier:'solo', cap:30, mrr:19.99, model:'claude-haiku-4-5-20251001'}, 'opposer-pro': {tier:'pro', cap:80, mrr:39.99, model:'claude-sonnet-4-6'}, 'opposer-team': {tier:'team', cap:200, mrr:89.99, model:'claude-sonnet-4-6'}, } Deno.serve(async (req) => { const form = await req.formData() const email = form.get('email')?.toString() const name = form.get('full_name')?.toString() const productSlug = form.get('product_permalink')?.toString() ?? '' const subId = form.get('subscription_id')?.toString() const saleType = form.get('sale_type')?.toString() // 'subscription_restarted', 'sale', etc if (!email) return new Response('no email', {status: 400}) const cfg = TIER_MAP[productSlug] if (!cfg) return new Response('unknown product', {status: 200}) const supabase = createClient( Deno.env.get('SUPABASE_URL')!, Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')! ) const now = new Date() const renewal = new Date(now) renewal.setMonth(renewal.getMonth() + 1) await supabase.from('subscribers').upsert({ email, name, tier: cfg.tier, status: 'active', run_cap: cfg.cap, model: cfg.model, mrr_usd: cfg.mrr, gumroad_subscription_id: subId, gumroad_product_id: productSlug, billing_month: `${now.getFullYear()}-${now.getMonth()+1}`, runs_used_this_month: 0, renewal_date: renewal.toISOString(), last_active: now.toISOString(), }, {onConflict: 'email'}) return new Response('ok', {status: 200}) })
Claude API Proxy Function
Replace the direct Anthropic call in the frontend with this proxy. Keeps your API key server-side and enforces run caps. Save as supabase/functions/oppose/index.ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2' import Anthropic from 'https://esm.sh/@anthropic-ai/sdk@0.20.9' Deno.serve(async (req) => { if (req.method === 'OPTIONS') return new Response(null, {headers:{'Access-Control-Allow-Origin':'*','Access-Control-Allow-Headers':'*'}}) const {email, docType, content, isTrial, systemPrompt} = await req.json() const supabase = createClient(Deno.env.get('SUPABASE_URL')!, Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!) const anthropic = new Anthropic({apiKey: Deno.env.get('ANTHROPIC_KEY')!}) // Check subscriber const {data: sub} = await supabase.from('subscribers').select('*').eq('email', email).single() if (!sub) return new Response(JSON.stringify({error:'not_found'}), {status:404}) if (sub.runs_used_this_month >= sub.run_cap) return new Response(JSON.stringify({error:'cap_reached'}), {status:429}) // Call Claude const msg = await anthropic.messages.create({ model: sub.model, max_tokens: 1200, system: systemPrompt, messages: [{role:'user', content}] }) // Increment run count await supabase.from('subscribers').update({ runs_used_this_month: sub.runs_used_this_month + 1, last_active: new Date().toISOString() }).eq('id', sub.id) // Log the run await supabase.from('run_log').insert({ subscriber_id: sub.id, doc_type: docType, model_used: sub.model, tokens_used: msg.usage.input_tokens + msg.usage.output_tokens, is_trial: isTrial }) return new Response(JSON.stringify({text: msg.content[0].text}), { headers: {'Content-Type':'application/json','Access-Control-Allow-Origin':'*'} }) })
